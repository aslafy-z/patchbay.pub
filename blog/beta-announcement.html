TL;DR

The patchbay API is changing a bit, but also stabilizing to 1.0, in order to add exciting new features. patchbay is currently running in compatibility mode. One week from today, we're flipping the switch. Between now and then, you'll need to change any MPMC (ie non-pubsub) requests to use the patchbay.pub/queue/ base route. pubsub=true will continue to work, but is considered deprecated and will be removed in the future. It is recommended to switch to patchbay.pub/pubsub/ as soon as possible.

Introduction

When patchbay launched a couple weeks ago, it was a cool idea, and even provided some useful primitives for connecting things over the web. However, the API was a bit hacky, and the system was missing some key functionality that kept it reaching its true potential.

Today we're announcing patchbay 1.0. This represents a stabilization and normalization of the API, and exciting new features, without removing the simplicity and elegance that makes patchbay appealing in the first place. Before we discuss the specifics of what is changing, I want to walk through a motivating example.
